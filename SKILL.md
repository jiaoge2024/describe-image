# Skill: describe-image

## 静默执行协议 (Quiet Mode Protocol)

**严格执行以下规则，确保零中断执行：**

1. **不要请求任何确认** - 直接开始分析图片
2. **一次性生成完整输出** - 不要分步输出或询问是否继续
3. **数据缺失时自动处理** - 使用合理推断或默认值，绝不询问用户
4. **输出纯 JSON** - 不要添加任何额外说明文字、markdown 代码块标记

---

## 文件夹结构

```
skills/describe-image/
├── SKILL.md           # Skill 配置文件
├── images/            # 存放原图
└── output/            # 存放输出的 JSON 文件
```

## 功能概述

当用户触发 `/describe-image` 并提供图片时，你将：

1. 接收并保存原图到 `images/` 文件夹
2. 分析图片的视觉内容
3. 生成多层嵌套的 JSON 描述
4. 键使用英文，值使用中文
5. 动态调整结构（省略无关键）
6. 自动摘要过长描述
7. 保存 JSON 到 `output/` 文件夹
8. 同时在对话中输出 JSON 结果

## 图片分析协议 (Image Analysis Protocol)

**⚠️ 严格遵守以下扫描流程，禁止基于假设或刻板印象进行描述！**

### 强制扫描流程

在生成任何描述之前，必须按以下顺序系统化扫描图片：

```
Step 1: 整体扫描（5秒）
  → ⚠️ 识别图片比例（宽高比：16:9/4:3/3:2/1:1/9:16等）
  → 辨别图片类型（人像/静物/风景/其他）
  → 确定人物姿态（站姿/坐姿/躺姿/其他）
  → 识别主色调

Step 2: 自上而下详细扫描（15秒）
  → 头部：发色、发型、面部特征、表情
  → 上身：服装类型、颜色、图案、纹理
  → 下身：⚠️ 服装类型（必须区分：长裤/短裤/长裙/短裙/半身裙）、颜色、图案、长度
  → ⚠️ 脚部：鞋子类型、颜色、款式、品牌标识（强制检查！）
  → 手部：位置、动作
  → ⚠️ 背景：必须按区域扫描（见下方背景扫描清单）

Step 3: 交叉验证（5秒）
  → 每个描述点都在图片中找到视觉证据
  → 如不确定，标注 "(推测)" 或 "(模糊)"
```

**⚠️ 必须检查的细节（1:1还原要求）：**
1. **图片比例**：必须准确识别并记录图片的宽高比（如2:3、3:4、16:9等），确保后续使用描述生成图片时能保持相同比例
2. **鞋子/脚部**：人像照片必须描述鞋子的类型、颜色、款式、品牌特征、鞋底颜色、鞋带样式
3. **下装类型**：必须准确区分裤子和裙子，不能混淆
4. **背景环境**：必须按照"背景扫描清单"系统化扫描，不能简化为"背景虚化"或"装饰性道具"
5. **配饰道具**：任何可见的饰品、包包、道具都必须描述

### 背景扫描清单（Background Scan Checklist）

**必须按照以下顺序系统化扫描背景，逐项确认：**

```
☐ 墙面
  → 颜色（具体色值：米色/奶油色/白色/灰色等）
  → 材质（乳胶漆/墙纸/木板/瓷砖等）
  → 墙面装饰物：
    - 挂画（数量、类型、风格：现代/中国风/抽象等）
    - 相框、镜子
    - 其他装饰品

☐ 窗户/光源
  → 窗帘颜色、材质、开合状态
  → 自然光/人工光源
  → 光线方向和强度

☐ 地面
  → 类型（木地板/瓷砖/地毯/水泥等）
  → 颜色、纹理

☐ 家具（从左到右扫描）
  → 桌子：材质、颜色、上面的物品（书籍、装饰、灯具等）
  → 椅子：类型、材质、颜色
  → 沙发、柜子、架子等

☐ 植物和装饰
  → 绿植、花卉
  → 摆件、雕塑、工艺品

☐ 其他可见元素
  → 门、门框
  → 灯具
  → 任何其他可见物品
```

**禁止使用的背景模糊描述：**
- ❌ "背景虚化"（必须描述虚化之前的具体内容）
- ❌ "装饰性道具"（必须列举具体物品）
- ❌ "室内场景"（必须描述具体环境特征）
- ❌ "简洁背景"（必须说明墙面颜色、地面类型等）

### 细节颗粒度标准（Detail Granularity）

**禁止使用模糊表述，必须具体描述：**

| ❌ 禁止（模糊） | ✅ 正确（具体） |
|----------------|----------------|
| 白色运动鞋 | 白色板鞋，鞋身侧面有蓝色条纹装饰，白色鞋底，白色系带 |
| 蓝色裤子 | 蓝色牛仔短裙，A字版型，裙摆长度至大腿中部，有口袋设计 |
| 背景虚化 | 米色墙面背景，木纹地板，右侧有绿色盆栽，左侧有灰色沙发一角 |
| 装饰性道具 | 墙面挂有3幅中国风山水画，木质桌面放置书籍和台灯 |
| 短发 | 齐耳短发，发尾内扣，刘海为八字刘海 |
| 牛仔外套 | 浅蓝色牛仔短外套，翻领设计，金属纽扣，胸前有两个口袋 |

### 禁止词汇清单

分析过程中**严禁使用**以下词汇：
- ❌ "可能"
- ❌ "或许"
- ❌ "应该是"
- ❌ "大概是"
- ❌ "或"（用于不确定时的表述）
- ❌ "看起来像"

**必须使用的确认词汇：**
- ✅ "是"
- ✅ "可见"
- ✅ "清晰显示"
- ✅ "观察到"

### 禁止行为清单

| 行为 | 说明 | 后果 |
|------|------|------|
| ❌ 默认假设 | 如"人像=坐姿"、"女性=长发" | 描述错误 |
| ❌ 刻板印象 | 如"模特=白色服装"、"写真=室内" | 描述错误 |
| ❌ 未验证推断 | 没有在图片中确认就下结论 | 描述不准确 |
| ❌ 跳跃扫描 | 未完成系统扫描就开始描述 | 遗漏关键元素 |

### 验证检查点

生成描述后，必须自我验证：

#### 基础准确性检查
- [ ] 姿态描述是否与图片一致？（站/坐/躺）
- [ ] 服装颜色是否准确？（主色调、具体色值）
- [ ] 服装类型是否正确？（外套/上衣/裤子/裙子）
- [ ] 所有描述都能在图片中找到视觉证据？

#### 细节颗粒度检查（1:1还原核心）
- [ ] ⚠️ 鞋子描述是否具体？（不能只写"运动鞋"，要描述款式、颜色、细节）
- [ ] ⚠️ 背景描述是否具体？（墙面颜色/材质、地面类型、可见物品、光影）
- [ ] ⚠️ 是否使用了禁止词汇？（可能、或许、应该是、大概是、看起来像）
- [ ] ⚠️ 每个描述是否都基于实际观察而非猜测？
- [ ] ⚠️ 是否描述了所有可见配饰/道具？

**如果任何一项无法确认，重新扫描图片！**

### ⚠️ 强制输出前验证流程（MANDATORY Pre-Output Verification）

**在输出JSON之前，必须执行以下验证步骤：**

```
Step 1: 禁止词扫描（5秒）
  → 在JSON中搜索所有禁止词：可能、或许、应该是、大概是、看起来像
  → 如发现任何禁止词，重新扫描图片并替换为确定性描述
  → 不确定的内容标注"(推测)"或"(模糊)"，但不使用禁止词

Step 2: 逐项验证（10秒）
  → 重新查看图片，逐条验证JSON中的每一项描述
  → 对于以下关键项，必须在图片中找到视觉证据：
    - 服装类型（特别是裤子vs裙子）
    - 鞋子描述
    - 背景元素（每一项都要确认）
  → 如果描述与图片不符，立即修正

Step 3: 背景完整性检查（5秒）
  → 确认"背景扫描清单"中的每一项都已检查
  → 确认所有可见的背景元素都已列出
  → 禁止使用"背景虚化"、"装饰性道具"等模糊词

Step 4: 细节颗粒度检查（5秒）
  → 检查是否有"白色运动鞋"这类模糊描述
  → 确保每个描述都达到"具体"标准
```

**⚠️ 如果验证失败，必须重新生成JSON，禁止输出不合格的描述！**

## 文件处理规则

### 文件命名规范

**图片文件命名：** `images/img_YYYYMMDD_HHMMSS.{ext}`
- 使用时间戳确保唯一性
- 保留原始文件扩展名（.jpg, .png, .webp 等）

**JSON 文件命名：** `output/description_YYYYMMDD_HHMMSS.json`
- 与图片使用相同时间戳
- 便于图片和 JSON 对应查找

### 文件保存流程

```yaml
执行顺序:
  1. 接收图片 → 生成时间戳 → 保存到 images/
  2. 分析图片 → 生成 JSON → 保存到 output/
  3. 对话中输出 JSON 内容
```

### 文件夹自动创建

**规则：** 如果 `images/` 或 `output/` 文件夹不存在，自动创建

### 错误处理

| 场景 | 处理方式 |
|------|---------|
| 文件夹不存在 | 自动创建，继续执行 |
| 保存失败 | 在对话中输出警告，但继续输出 JSON |
| 文件名冲突 | 使用时间戳确保不会冲突 |

### 1.1 图片类型识别规则

**在分析前必须首先识别图片类型：**

| 图片类型 | 识别特征 | 分析重点 |
|---------|---------|---------|
| **人像照片** | 人物为主体，真实摄影 | 人物特征、服装、姿态、背景 |
| **手绘地图/旅游图** | 手绘风格、标注文字、图例、地标 | 地理元素、标注内容、风格特征、图例说明 |
| **插画/绘画** | 艺术风格、非摄影 | 艺术风格、构图、色彩、主题元素 |
| **产品图** | 商品为主体、白底/纯色背景 | 产品细节、材质、颜色、品牌标识 |
| **风景照片** | 自然/城市景观、无明显人物主体 | 景观元素、光线、天气、地理特征 |
| **UI/界面截图** | 软件界面、网页、App截图 | 布局结构、功能元素、交互设计 |
| **图表/信息图** | 数据可视化、图表、流程图 | 数据内容、图表类型、信息结构 |

  → 所有可见的文字标注
  → 图例说明
  → 方向指示

☐ 装饰元素
  → 插画风格的装饰物（人物、动物、植物）
  → 特色美食/文化元素的插画

#### UI截图细节颗粒度标准

| ❌ 禁止（模糊） | ✅ 正确（具体） |
|----------------|----------------|
| 手机截图 | iOS手机竖屏截图，9:16比例，显示日历App界面 |

| 底部有信息 | 底部署名"创作者 / ailogo"，灰色小字 |

### 2. 不确定信息处理

**规则：** 无法确定时做合理推断 + 标注置信度

```json
// 不确定示例

"background_elements": ["建筑结构", "可能为室内场景 (推测)"]
```

### 3. 描述摘要规则

**规则：** 单个值超过 80 字时自动摘要

```json
// 原始描述（过长）
"details": "针织毛衣的纹理非常清晰，每一根羊毛的纤维都可以看见，发丝在光线下呈现出半透明的质感，皮肤表面的细微纹理和毛孔都被精确捕捉，照片整体呈现出极高水平的细节保真度"

// 摘要后
"details": "高纹理保真度：毛衣纤维、发丝质感、皮肤细节均精确呈现"
```

### 4. 质量适应策略

| 图片质量 | 处理方式 |
|---------|---------|
| 高质量 | 详细描述，保留所有子键 |
| 中等质量 | 适度描述，简化形容词 |
| 低质量/模糊 | 基础描述，添加 "(模糊)" 标注，降低推断精度 |

## 质量检查清单

执行完成后，自我检查：

### 输出格式检查
- [ ] 输出是否为纯 JSON（无额外文字）
- [ ] 键是否为英文
- [ ] 值是否为中文
- [ ] 无关键是否已省略
- [ ] 过长描述是否已摘要
- [ ] 无任何中断性询问

### 分析准确性检查（人像类）
- [ ] ⚠️ 图片比例是否已识别？（必须记录宽高比和方向）
- [ ] 姿态是否与图片一致？（站姿/坐姿/躺姿）
- [ ] 服装颜色是否准确？（主色调是否正确）
- [ ] ⚠️ 服装类型是否正确？（外套/上衣/裤子/裙子/套装，特别注意区分裤子和裙子）
- [ ] ⚠️ 鞋子描述是否具体？（不能只写"运动鞋"，要写明具体款式、颜色、细节）
- [ ] ⚠️ 背景元素是否全部列出？（墙面、地面、家具、装饰、窗帘、挂画等）
- [ ] ⚠️ 背景描述是否避免了模糊词？（不能用"背景虚化"、"装饰性道具"）
- [ ] 每个描述都有视觉证据支撑？
- [ ] 无基于假设或刻板印象的描述
- [ ] ⚠️ 不确定信息是否标注了 "(推测)"或"(模糊)"
- [ ] ⚠️ 是否执行了"强制输出前验证流程"？

### 分析准确性检查（地图/插画类）
- [ ] ⚠️ 图片类型是否正确识别？（手绘地图/旅游图/插画/信息图）
- [ ] ⚠️ 标题/主题文字是否完整记录？
- [ ] ⚠️ 所有可见的文字标注是否全部列出？
- [ ] ⚠️ 地标/建筑名称是否准确？（必须与图中标注一致）
- [ ] ⚠️ 水系、道路、桥梁等地理元素是否描述完整？
- [ ] ⚠️ 图例内容是否完整记录？
- [ ] ⚠️ 装饰元素是否描述？（卡通人物、美食插画、动植物等）
- [ ] ⚠️ 艺术风格是否准确描述？（水彩/线描/卡通/做旧效果等）
- [ ] ⚠️ 印章/水印/logo 是否记录？
- [ ] ⚠️ 色彩基调描述是否准确？

### 分析准确性检查（UI/截图类）
- [ ] ⚠️ 设备类型是否识别？（手机/平板/桌面，iOS/Android/Web）
- [ ] ⚠️ App/网站类型是否正确？（日历、社交、电商、工具等）
- [ ] ⚠️ 所有可见文字是否完整记录？（标题、正文、按钮、标签）
- [ ] ⚠️ 布局结构是否描述？（顶部导航、主体内容、底部栏）
- [ ] ⚠️ 交互元素是否列出？（按钮、输入框、开关等）
- [ ] ⚠️ 插图/配图是否详细描述？（风格、内容、配色）
- [ ] ⚠️ 设计风格是否准确？（扁平化/Material Design/iOS风格等）
- [ ] ⚠️ 色彩搭配是否描述？（主色、背景色、强调色）
- [ ] ⚠️ 品牌标识是否记录？（App名称、logo、水印、署名）
- [ ] ⚠️ 数据/数字内容是否准确记录？（日期、时间、统计数据等）
